# MDX

本ページは、テンプレート言語として MDX (と JSX) を採用した理由と、その挙動を支えている `@tuqulore-inc/eleventy-plugin-preact` の役割を扱います。

## MDX を採用した理由

本ボイラープレートは、ページ本体を書くフォーマットに MDX を選んでいます。Markdown だけでは足りないケース (動的なコンポーネント、Partial Hydration の例、共通コンポーネントの差し込み) で、テンプレート言語を差し替える必要をなくすためです。

Markdown で 8 割の本文を書き、残りの 2 割で JSX に落ちる形は、記事主体のサイトと、コンポーネント主体のサイトの中間に位置します。docs、コーポレートサイト、ランディングページのいずれでも同じ書き方で通せます。

Markdown と JSX を別々に扱うと、レイアウトチェイニングや共通データの参照が言語をまたぐことになります。MDX に統一することで、レイアウトから本文まで同じ Preact コンポーネントの世界で扱えるようになります。

## 何を SSR しているのか

`@tuqulore-inc/eleventy-plugin-preact` は、Eleventy のテンプレート形式として `.jsx` と `.mdx` を登録し、その両方を Preact のサーバサイドレンダラで HTML に変換します。JSX と MDX は Node.js のカスタムローダを通してモジュールとして読み込まれるため、テンプレートファイル自体を通常の import と同じ規則で扱えます。

サーバ側で完結する処理はここまでです。クライアント側の JavaScript は Partial Hydration の層 ([`preact-island`](/ja/architecture/preact-island/)) が担い、SSR プラグイン自体は「HTML を返すだけ」の関心に閉じます。

## `export const data` でページデータを渡す

MDX ファイルの先頭で `export const data = { ... }` と書くと、その値が Eleventy のページデータになります。Markdown の frontmatter と等価な仕組みです。

```mdx
export const data = {
  layout: "post",
  title: "はじめてのサイト",
};

# はじめてのサイト
```

Markdown の YAML frontmatter を書き足すと JSX と競合するため、MDX では `export` に統一しています。JSX ファイル (`.jsx`) でも同じ書き方でページデータを渡せます。

## `eleventy` シングルトンで prop drilling を避ける

サーバサイドレンダリング中に限り、`@tuqulore-inc/eleventy-preset/eleventy` から `eleventy` シングルトンを import できます。

```jsx
import { eleventy } from "@tuqulore-inc/eleventy-preset/eleventy";

<title>
  {eleventy.title} | {eleventy.site.name}
</title>;
```

`eleventy.site` (グローバルデータ)、`eleventy.title` (ページデータ)、`eleventy.content` (子テンプレートから受け取った HTML)、`eleventy.page` (URL や日付) などがまとめて取れます。partial から使えば、レイアウトから props を渡していく必要がありません。

このシングルトンは SSR 時にだけ有効です。クライアント側で hydrate されるコンポーネント (Island) からは触れません。Island に渡したい値は、`<Island>` の props として明示的に渡すのが規約です。

## レイアウトチェイニングと組み合わせる

MDX のページは `export const data.layout` で親レイアウトを指定できます。親レイアウトも MDX で書け、さらにその親を指定できます。scaffold の default テンプレートは `index.mdx → post.mdx → base.mdx` の 3 段構成で、`post` が記事のラッパ、`base` が HTML 全体の骨格を担います。

各レイアウトは `eleventy.content` を `dangerouslySetInnerHTML` で埋め込みます。子テンプレートの描画結果を親レイアウトの JSX ツリーに差し込むための決まりです。

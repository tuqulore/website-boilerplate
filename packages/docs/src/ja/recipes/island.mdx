# Island の粒度のレシピ

本ページは、Island (Partial Hydration) を使うかどうかの判断と、使うときの粒度の目安を扱います。Island の内部設計は [Architecture / Partial Hydration](/ja/architecture/preact-island/) を参照してください。

## Island を使わない選択肢を先に考える

Island は「クライアント側の JavaScript が必要な要素だけを部分的に hydrate する」仕組みです。逆に言えば、JavaScript が要らない要素には使いません。次のいずれかで済む場合、Island を使うより単純です。

- `<details>` / `<dialog>` などの HTML 要素で完結する UI (アコーディオン、モーダル)
- CSS の `:target` や `:has()` セレクタで表現できる状態遷移
- リンクとページ遷移で済むフロー

「JS がないと本当に困るか」を先に検討します。docs サイトのように読者が触るのは検索とナビゲーションのごく一部、というケースでは、Island の対象は自然と絞られます。

## 何を Island 化するか

Island 化の判断基準はシンプルです。

- **状態を持つ**: カウンター、フォーム、フィルタ UI など。
- **外部データを読む**: 検索、フェッチ、遅延読み込み。
- **ユーザー操作で描画が変わる**: メニュートグル、タブ切替。

このいずれにも当てはまらない要素 (ロゴ、静的なテキスト、リンク集など) は Island 化する必要がありません。SSR された HTML で完結します。

## 粒度を細かくするか、粗くするか

Island の単位を細かくすると、hydrate 対象の JS が小さくなります。ヘッダのメニューだけ、フッタのニュースレター登録フォームだけ、というふうにページの一部だけを Island 化するのが原則です。

一方で、隣接した複数の UI が同じ状態を共有する場合 (フィルタとリストが連動するなど) は、その組を 1 つの Island としてまとめます。Island を跨いだ状態共有は、素の Preact のプロップスや Signals では表現できないためです。

粒度の判断で迷ったら、「同じ状態を触る要素は同じ Island に入れる」を目安にします。

## トリガーを選ぶ

`<Island>` の `on` 属性で、hydrate のタイミングを切り替えられます。

| 値            | 使い所                                                                               |
| ------------- | ------------------------------------------------------------------------------------ |
| `interaction` | クリック・フォーカスがあるまで待つ。多くのボタンやフォームに向く。デフォルト値。     |
| `visible`     | ビューポートに入ったら hydrate。ページ後半に置いたインタラクティブなブロックに向く。 |
| `idle`        | ブラウザがアイドルになったら hydrate。初期表示直後に必ず動かしたい軽量な UI に向く。 |

判断基準は「利用者がその Island に触るまでにどれくらい猶予があるか」です。触るまで時間がある要素ほど遅らせ、初期表示直後に必要な要素ほど早めに hydrate します。

パラメータ付きトリガー (`on:media("(min-width: 40rem)")` など) が必要な場合は、`<Island>` を使わずに素の `<is-land>` を直接書きます。詳細は `@tuqulore-inc/eleventy-plugin-preact-island` の README を参照してください。

## SSR と一致させる

`<Island>` に渡した props は SSR 側の描画とクライアント側の hydrate の両方に流れます。SSR で表示された内容と hydrate 後の内容が一致することが保証されます。

これを守るために、Island の中で `new Date()` や `Math.random()` のような不確定な値を初回描画に使わないでください。SSR とクライアントで異なる値が生成され、hydrate 時に mismatch が起きます。初回描画は決定的な値だけを使い、動的な値は `useEffect` の中で反映するようにします。

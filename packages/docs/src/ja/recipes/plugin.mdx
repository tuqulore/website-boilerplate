# プラグインを追加するレシピ

本ページは、Eleventy の外部プラグイン (RSS、Syntax Highlight、ナビゲーションなど) を、本ボイラープレートのプリセットと共存させて追加する手順を扱います。

## 依存を追加する

まずパッケージを `devDependencies` に加えます。

```shell
pnpm add -D @11ty/eleventy-plugin-rss
```

`peerDependencies` を持つプラグインの場合は、指示に従って必要な peer も入れます。

## `preset()` の extend から `addPlugin` する

`eleventy.config.js` のプリセット呼び出しに関数を渡し、その中で `eleventyConfig.addPlugin` を呼びます。

```js
import preset from "@tuqulore-inc/eleventy-preset";
import pluginRss from "@11ty/eleventy-plugin-rss";

export default preset((eleventyConfig) => {
  eleventyConfig.addPlugin(pluginRss);
});
```

extend は、プリセットが自身の 3 プラグイン (preact、preact-island、postcss) を登録した後に呼ばれます。プリセットの設定を土台にして、その上に追加のプラグインやフィルタを積む形です。

## 既存プラグインと重複しないか確かめる

追加するプラグインが Eleventy の設定を上書きする種類のもの (`setInputDirectory`、`setOutputDirectory` を呼ぶ、`addTemplateFormats` で `css` を再登録するなど) の場合、プリセット側の設定と衝突する可能性があります。プラグインの README で何を触っているかを確認してください。

たとえば `setInputDirectory` を呼ぶプラグインは、プリセットの `src` 固定と競合します。`setOutputDirectory` も同様です。この場合は、プラグインが既定の入出力ディレクトリに追従する設計になっているかを確認するか、プリセットを使わずにプラグインを直接組む選択肢を検討します。

## Eleventy の設定を触るフィルタ・shortcode の追加

小さな拡張 (フィルタ、shortcode、コレクションの追加) は、プラグインを介さずに extend の中で直接書けます。

```js
export default preset((eleventyConfig) => {
  eleventyConfig.addFilter("upper", (value) =>
    String(value ?? "").toUpperCase(),
  );

  eleventyConfig.addShortcode("year", () => new Date().getFullYear());

  eleventyConfig.addCollection("posts", (collection) =>
    collection.getFilteredByGlob("src/**/*.mdx").reverse(),
  );
});
```

Eleventy の [`UserConfig` API](https://www.11ty.dev/docs/config/) をそのまま使えます。

## 独自プラグインを切り出したくなったら

extend に書いた関数が長くなったら、独立したプラグインとして切り出します。プラグインは関数形式で書き、`eleventyConfig` を第 1 引数に受け取ります。

```js
// src/plugins/my-plugin.js
export default function myPlugin(eleventyConfig, options = {}) {
  eleventyConfig.addFilter("upper", (value) =>
    String(value ?? "").toUpperCase(),
  );
}
```

```js
// eleventy.config.js
import preset from "@tuqulore-inc/eleventy-preset";
import myPlugin from "./src/plugins/my-plugin.js";

export default preset((eleventyConfig) => {
  eleventyConfig.addPlugin(myPlugin, { verbose: true });
});
```

プラグインとして切り出しておくと、テストしやすくなり、他のプロジェクトへの持ち出しも簡単です。Eleventy 公式の [プラグイン作成ガイド](https://www.11ty.dev/docs/plugins/) が詳しい参考になります。
